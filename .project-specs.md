## 2. Stack Tecnológico Estricto (Vercel-Native)
* **Framework Frontend:** Next.js 15 (App Router).
* **Framework Backend / CMS:** Payload CMS 3.0 (Integrado vía `@payloadcms/next/utilities`).
* **Base de Datos:** PostgreSQL (preparado para Vercel Postgres).
* **Estilos:** Tailwind CSS puro. (Prohibido crear archivos CSS personalizados, usar utilidades).
* **Manejo de Formularios/Emails:** Utilizar el servidor SMTP propio del cliente para enviar los leads: Servidor `mail.vypersolutions.cl`, Puerto `465`, Cifrado `SSL/TLS`.

## 3. Sistema de Diseño (UI/UX)
* **Paleta de Colores:** * Fondo Principal: Dark Mode industrial (Negros y grises oscuros como `bg-zinc-950` o `bg-slate-900`).
  * Color de Acento (Call to Action): Azul vibrante/eléctrico (basado en el logo corporativo).

* **Tipografía:** Sans-serif limpias y modernas (ej. Inter, Montserrat) para lectura, y fuentes audaces para los H1/H2.

## 4. Arquitectura de Base de Datos (Payload Collections)
Agentes de IA, deben configurar los siguientes esquemas en TypeScript:
1. `Services` (Servicios):
   * `title` (Text).
   * `slug` (Text).
   * `division` (Select: "Industrial" o "Motorsport").
   * `short_description` (Textarea).
   * `content` (RichText).
2. `Projects` (Portafolio de Casos de Éxito):
   * `title` (Text).
   * `service_id` (Relationship -> Services).
   * `image_before` (Upload).
   * `image_after` (Upload).
   * `description` (RichText).
3. `Leads` (Formulario de Contacto):
   * `name` (Text).
   * `email` (Email).
   * `phone` (Text).
   * `service_interest` (Select).
   * `message` (Textarea).

## 5. Reglas de Programación Front-End para la IA
* **Server Components:** Priorizar React Server Components para garantizar la indexación instantánea de Google. Solo usar `'use client'` en componentes interactivos (como el menú móvil o el formulario de contacto).
* **Optimización de Media:** Las imágenes del taller (chispas, soldaduras, motores) deben renderizarse usando exclusivamente el componente `<Image />` de `next/image`.
* **Estructura Semántica:** Usar etiquetas semánticas (`<header>`, `<section>`, `<article>`).
* **Microdatos SEO:** Inyectar un script `application/ld+json` en el layout principal definiendo la empresa como `LocalBusiness` y `AutoRepair`, incluyendo la dirección física de Coquimbo.
